# Server Feature Validation Matrix

이 문서는 **Neon Survival Prototype & E2E Verification Harness** 개발 주기에 따라 AI Game Company Server의 핵심/보안 기능들이 어떻게 검증되는지 정리한 상세 매트릭스입니다.

| Server Feature | Validation Task | Evidence | Expected Result | Failure Signal | Manual Review Needed |
| --- | --- | --- | --- | --- | --- |
| **Project / Epic / SubEpic / Task Hierarchy** | Task 1: Project Bootstrap | SQLite database records inside `projects`, `epics`, `sub_epics`, `tasks` tables. | All hierarchy tiers are successfully populated with correct foreign key constraints. | `sqlite3.IntegrityError` or empty table select outputs. | No (Automated check) |
| **Owner Task Planning Validator** | Task 1 (Pre-execution) | Planning Validator stdout log matching schema validation. | The JSON structure passes task size and evidence requirements check with code 0. | `ValidationError` in `app/owner_task_validator.py`. | No (Automated check) |
| **Workspace Worker branch / commit / report flow** | Tasks 1 - 8, 10 | Git branch metadata (`worker/*`) and API `/tasks/{id}/report` payloads. | Worker leases task, creates isolated branch, pushes commits, and reports execution status. | Commit push fails or API returns lease expiration/invalid branch name. | No (Automated check) |
| **Test Runner setup / test / smoke phases** | Tasks 1 - 8, 10 | `.game-company/test_runner.json` execution outputs. | Test runner sets up virtual environment, passes unit tests, and captures screenshot under headless dummy display. | Setup dependency error, pytest failure, or screenshot image generation error. | No (Automated check) |
| **Artifact upload** | Tasks 8, 10 | Files inside local `artifacts` directory and `artifacts` table records. | Raw screenshot and test log uploaded successfully using stream endpoints. | `HTTP 413` limit exceeded, empty files, or missing file paths on server. | No (Automated check) |
| **Artifact important / release classification** | Task 10: Approval / Release Candidate | SQLite DB artifact metadata column values (`important = 1`, `release_or_milestone = 1`). | Specific artifacts are flagged as important/release files. | `important = 0` or missing metadata columns for release files. | Yes (Verify flags match RC spec) |
| **Merge Review Policy warning / block behavior** | Task 9: Merge Policy Challenge | Synthetic test fixture validation reports and blocked merge responses. | The Merge Review Policy blocks the merge request because of missing test log evidence. | Merge succeeds despite missing or failing test evidence files. | Yes (Confirm block was triggered correctly) |
| **Approval API** | Task 10: Approval / Release Candidate | DB approval status transition (`pending` -> `approved`) and API `/approvals/{id}/decision` call logs. | Approval status changes to approved, enabling git branch merge. | Invalid status transition or unauthorized merge exception. | No (Automated check) |
| **Discord natural-language approval safety** | Task 10: Approval / Release Candidate | Gateway console logs and mock discord message event triggers. | Discord bot parses positive intent ("승인", "좋아 진행해") and issues decision payload. | Gateway parses ambiguous intent as valid or throws Discord connection errors. | Yes (Verify Gateway decision parsing) |
| **Memory refs** | Tasks 2 - 11 | Prompt context logs showing `memory_refs` tags mapping. | Enforces correct coding/project guidelines inside task assignment context. | Empty memory context or missing tags mapping during lease. | No (Automated check) |
| **Task history summaries** | Tasks 2 - 11 | DB task history records and prompt generation payloads. | Success summaries of preceding tasks are automatically injected into the next lease prompt. | Prompt generation payloads are missing historical task context. | No (Automated check) |
| **Artifact cleanup dry-run** | Task 11: Artifact Cleanup / MCP Permission Dry-run | Script execution output containing dry-run logs. | Only expired, non-important artifacts are candidate targets, leaving release files intact. | Disk files are unlinked in dry-run mode or database records are removed. | Yes (Verify dry-run candidate outputs) |
| **MCP permission skeleton dry-run** | Task 11: Artifact Cleanup / MCP Permission Dry-run | Permissions module test runs and console outputs. | Path validation blocks outside roots or system files (e.g. `.env`, `.git/config`). | `validate_mcp_call` returns `is_allowed = True` for outside directories. | Yes (Verify dry-run permission rejection logs) |
| **README / portfolio devlog generation** | Task 11: Artifact Cleanup / MCP Permission Dry-run | `docs/DEVELOPMENT_LOG.md` and updated `README.md` files. | Devlog summarizes all E2E verification statistics and successful test metrics. | Devlog is empty or missing E2E validation summaries. | Yes (Review final portfolio presentation) |
